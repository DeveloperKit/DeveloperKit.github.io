If you want to extract information from online or from an external database (eg. Twitter, Google, Facebook, etc.) you can do it two ways: Web Scraping or using an API.

WEB SCRAPING: Downloading a website’s contents/source, or more crudely put - ripping a everything off a web page.

** Using Pattern (for python)

x = URL('http://site.com').download()

** Note 1: this is quite literally “hacking” and may violate the terms of service of a company (thus making web scraping a particular website/app illegal). We do not endorse web scraping in this context, and most companies which have useful information provide an API to access it.

** Note 2: If you choose to scrape a web page, you must be prepared to deal with parsing out the gobblygook “stuff of websites”, namely: HTML (structure), CSS (style), and JS (scripts). There are many open-source libraries (pre-built set of functions you can use for free) out there that will parse out HTML, but not necessarily embedded CSS and JS. Likewise, many libraries will build the web contents into a “DOM Tree.” We encourage you to become more familiar with website structure and the DOM Tree before trying to extract information - but not always a requirement.

API: Application program interface (API for short). Defined by Wikipedia as set of routines, protocols, and tools for building software. Alternatively an interface for accessing other programs, databases, etc. without having direct access

** Note 1: Most API have documentation, though most of them are not informative nor easy to understand nor execute the way they say they will. You must be prepared to spend a few hours (perhaps days, depending on the API) trying to “figure things out” - no matter how much of a seasoned programmer you are. Github for example (yes, it has an API) has beautiful documentation - and still takes a long time to set up.

**Note 2: Example of the API we use is Twitter via Pattern. Twitter has an API which communicates with the Pattern API for Twitter. You can read the basic documentation of use here: http://www.clips.ua.ac.be/pages/pattern-web#twitter
